# 自建节点说明

有了订阅链接生成工具后，其实节点其实已经简化成了`vless+ProxyIP`，决定自己参考各大佬的代码，自己实现一个节点
1. 学习`JavaScript Streams API`
2. 简化定制代码，应当不容易报`1101`错误（一定学习`https://blog.cmliussss.com/p/CF1101/`）
3. 需要泄漏`uuid`和`sni`才能使用。（`vless+reality`其实有泄漏风险）
4. `path=/`需要知道格式才能享受`ProxyIP`否则体验太差
6. 所以完全不需要做那么多安全检查，效率能大大提升

## 如何本地测试？
1. `v2rayN`真连测试地址设置`http://www.msftconnecttest.com/connecttest.txt`
2. 系统代理：`自动配置系统代理`,路由：`全局`
3. 导入链接：`vless://12345678-1234-1234-1234-123456789012@127.0.0.1:8787?encryption=none&security=none&sni=127.0.0.1&fp=random&type=ws&host=localhost&path=%2F#LocalTest`
4. `npm run dev`

，先让AI实现了一版，本地真连测试`-1`。一步步检查，发现几个关键点：
1. `request.headers.get('sec-websocket-protocol')`，首包居然是通过`headers`获取的
2. `ws.send(new Uint8Array([0, 0])`，需要回复

测试成功后，又遇到非常多的异常，加了`try catch`后代码就膨胀了。还发现在`v2rayN`中内核由`Xray`换成`singbox`后，真连测试能过，但无法上网，未知原因。

## 如何CF节点测试?
1. 本地测试成功后，将代码上传到`Pages`, 记录下地址`xxx.pages.dev`
2. 导入链接：`vless://12345678-1234-1234-1234-123456789012@www.wto.org:443?encryption=none&security=tls&sni=xxx.pages.dev&fp=firefox&insecure=1&allowInsecure=1&type=ws&host=xxx.pages.dev&path=%2F#PagesTest`
3. 与本地测试的区别是多了`tls`相关参数，以及改了`host`和`sni`

## `ProxyIP`
cmliussss大佬提供了两套`ProxyIP`域名，例如：
1. 按数据中心：`hkg.proxyip.cmliussss.net`、`lax.proxyip.cmliussss.net`等
2. 按地区：`proxyip.hk.cmliussss.net`、`proxyip.us.cmliussss.net`、`proxyip.jp.cmliussss.net`等

大佬给的教程是用的按地区，但兜底代码是用的按数据中心。

所以我改了`vless_中文反代版.js`,使用`/?ip=colo.proxyip.cmliussss.net:443`,将自动替换`colo`为当前访问的[数据中心](https://www.cloudflarestatus.com/)
既，将使用域名下绑定的`A/AAAA`记录去连固定的VPS。而VPS中设置了`iptables`规则，将某个端口的所有请求转发到`CF CDN`中某个IP的端口

可以使用`https://site.ip138.com/lax.proxyip.cmliussss.net/`,查看登记的IP。也可以使用`nslookup -A hkg.proxyip.cmliussss.net`

## 搭建属于自己的cloudflare优选反代IP和ProxyIP
容易混淆的地方，根据甬哥的视频教程，区别如下
1. 优选反代IP：`v2rayN`中`address`填写的地方。它可以是CF的IP,也可以是非CF的IP
2. ProxyIP：`v2rayN`中`path`中`/?ip=xxx`填写的地方。必须是非CF的IP

它们使用`http://ip:port/cdn-cgi/trace`访问，显示400错误有`cloudflare`，使用`https://domain:port/cdn-cgi/trace`访问，显示`trace信息`

ProxyIP有两种搭建方案
1. iptables系列：https://blog.tanglu.me/cloudflare_proxy_ip
2. vless+reality: https://ygkkk.blogspot.com/2023/07/cfworkers-vless.html。教程八

推荐使用`https://github.com/yonggekkk/x-ui-yg`。只要将`reality`的域名设置成`time.is`、`ip.sb`等域名，就是`ProxyIP`，否则只是普通的`vless`节点。

域名是未被屏蔽的CF托管的官方域名，例如优选域名中各大厂的域名。

注意：PrxoyIP都有被扫，然后被分享的可能。所以，不要使用常见端口，推荐使用IPv6，时常关注VPS流量消耗。
