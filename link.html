使用指南

文本

直接返回内容中的文本内容。适合用于分享纯文本信息，如说明、代码片段、联系方式等。

跳转

由浏览器进行307跳转到目标地址。注意：部分软件不支持307跳转，此时请选择"代理"类型。

代理

由服务器爬取目标页面内容并转发给用户。适合需要获取远程内容的场景，兼容性好。速度略慢

API 操作说明 ▼

更新单个链接

您可以通过 API 直接更新单个链接的内容和类型，而无需通过网页界面。

1. 使用文件内容只更新 note 字段

curl -X PUT "https://your-domain.com/s/abc123`" \ -H "Authorization: Bearer API_KEY" \ -H "Content-Type: text/plain" \ --data-binary "@/path/to/file.txt"

2. 使用 JSON 更新请求

curl -X PUT "https://your-domain.com/s/abc123" \ -H "Authorization: Bearer API_KEY" \ -H "Content-Type: application/json" \ -d '{"note": "只更新note"}'
curl -X PUT "https://your-domain.com/s/abc123" \ -H "Authorization: Bearer API_KEY" \ -H "Content-Type: application/json" \ -d '{"note": "还更新type", "type": "redirect"}'
注意事项：
  • 将 https://your-domain.com 替换为您的实际域名
  • 将 /s/abc123 替换为您要更新的具体链接路径
  • 将 API_KEY 替换为您的实际 API 密钥
  • JSON 请求支持更新 note 和 type 字段
  • type 字段可选值: text, redirect, proxy

批量操作 API

本网页界面使用了以下 API 进行批量操作：

获取所有链接 (GET)

curl -X GET "https://your-domain.com/link/api" \ -H "Authorization: Bearer API_KEY"

保存所有链接 (POST)

curl -X POST "https://your-domain.com/link/api" \ -H "Authorization: Bearer API_KEY" \ -H "Content-Type: application/json" \ --data-binary "@/path/to/file.json"
关于 POST 请求的数据文件：
  • file.json 应包含所有链接数据的 JSON 数组
  • JSON 格式示例：[{"pathname": "/s/abc123", "note": "https://example.com", "type": "redirect"}]
  • 可以通过本页面的"下载备份"功能获取当前的 JSON 数据格式
